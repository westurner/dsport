## Strategic Mask: Generated Lexers

### Decision: Exclude `src/lexers/generated/` from Coverage Goals

**Rationale:**
1. **Auto-Generated Code**: 436+ lexer files generated by `tools/gen_lexer.py` from upstream Pygments
2. **Maintenance Burden**: Coverage gaps will be re-introduced with each upstream update
3. **Low ROI**: Fixing branch coverage in transpiled code provides minimal value
4. **Quality Preserved**: Existing byte-parity tests validate correctness against Python
5. **Better Focus**: Resources better spent on hand-written code

**Documentation Added**: [BRANCH_COVERAGE_IMPROVEMENTS.md](BRANCH_COVERAGE_IMPROVEMENTS.md) now includes detailed notes on masking strategy

## Lessons Learned

1. **API Constraints**: Lexer lookup via public API not available; tests focused on direct formatter API instead
2. **Test Design**: Simpler unit tests targeting concrete functions more effective than integration tests through opaque wrappers
3. **Markup Formatters**: Comprehensive escaping logic; 57 tests necessary for branch coverage of character-by-character processing
4. **Generated Code Strategy**: Masking generated code as "high-maintenance, low-ROI" aligns better with reality than chasing coverage
